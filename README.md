## Commit conventions
Commit structure I will be using in the project

1. feat:       New feature                     
2. fix:        Bug Fixes             
3. docs:       Documentation                        
4. style:      Format (without changing the logic)          
5. refactor:   Rewrite code without changing the logic
6. test:       Tests                               
7. chore:      Maintenance tasks
